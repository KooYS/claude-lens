…/server/mcp-server.js

Troubleshooting

Server keeps showing "Not running" after install

Run ./native-host/install.sh from the project root, then click Connect. The server starts on connect, not on panel open.

Auto-start doesn't work (native messaging failed)

Chrome Guest profile blocks native messaging. Switch to a regular Chrome profile, or start the server manually: npm start --prefix ~/path/to/claude-lens/server

MCP tools show "failed" in Claude Code

After running install.sh, fully restart Claude Code — the new MCP config won't load until then.

Claude CLI shows "Not found"

Re-run install.sh while Claude CLI is installed, or click the Claude CLI field to set the path manually.

MCP server path not found / MCP path shows placeholder

The MCP path is registered in ~/.claude/settings.json by install.sh. If missing, re-run install.sh and check that the file contains a claude-lens entry under mcpServers.
